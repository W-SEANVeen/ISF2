## 摘要

《孤城》(The Isolated Fortress) 是一款基于 **Unity 2022 LTS + PICO 3 (移动 VR)** 平台开发的 VR 历史仿真体验作品。项目取材自东汉永平十八年（公元 75 年）耿恭疏勒城保卫战——"十三人归玉门"的真实历史事件，旨在通过 **零摇杆位移、真实物理运动** 的具身交互方式，结合极简的 **黑白红水墨视觉风格（NPR 非真实感渲染）**，构建高压的攻城防御体验。

项目采用事件驱动的战斗指挥架构，以 **BattleDirector** 为总控核心，集成了多路步兵方阵阵型维护（SquadCommander）、精英死士有限状态机（EnemyAssault — 7 状态 FSM）、箭雨物理系统（250 对象池 + 抛物线弹道）、全局阻尼压制机制（SiegeManager）、骑兵冲锋（EnemyHorse）以及大黄弩 VR 射击（CrossbowShoot）等核心玩法系统。视觉上使用 URP 管线配合自研手写 HLSL Shader（InkCodeShader 水墨透明效果、FlagWind 旗帜飘动顶点动画）和 Shader Graph（SG_Terrain 地形混合、SG_PropOutline 法线描边），实现移动端高性能的水墨风格渲染。

项目已通过 PICO 3 真机验证，解决了 PICO SDK 3.4.0 → 3.1.0 降级的兼容性问题，APK 可正常运行。作品具备数字文旅、体感教育、全民健身等多元价值延伸潜力。

**关键词**：VR 历史仿真；具身交互；水墨渲染；疏勒城；Unity；PICO

## 二、运行方法

### 2.1 运行前提

1. **Unity 编辑器运行**：需安装 Unity 2022 LTS，打开项目后确保已导入以下 Package：
   - Universal RP（URP）
   - XR Interaction Toolkit
   - PICO Unity Integration SDK v3.1.0
   - AI Navigation（NavMesh）
2. **PICO 真机运行**：需将项目打包为 Android APK，安装至 PICO 3 设备。打包前确认：
   - Build Settings → Platform 切换为 Android
   - Player Settings → Minimum API Level ≥ 29
   - PXR_ProjectSetting 配置正确

### 2.2 运行流程

```
BootScene（启动加载）→ GameStartMenu（主菜单）
    ↓ 点击"开始游戏"
Scene2_city_v2（战斗场景）
    ↓ 3 秒后战斗自动启动
    ↓ 玩家死亡或被死士击杀
GameEnd（结算场景，文字滚动展示结局）
    ↓ 点击"返回"
GameStartMenu（循环）
```

### 2.3 操作说明

| 操作 | 手柄动作 | 效果 |
|------|---------|------|
| 大黄弩射击 | 扣动扳机键（Trigger） | 箭矢沿枪口方向飞出，命中敌人触发全局压制 |
| 抓取武器 | 按住 Grip 键 | 拾取/切换武器 |
| 躲避箭雨 | 真实下蹲 / 侧移 | 被敌箭命中会闪红 + 短暂僵直 |
| 暂停 | 按下手柄 Menu 键 | 呼出暂停菜单，射线点击 UI |

### 2.4 构建产物

打包后的 APK 文件可直接安装至 PICO 3 设备运行。如需在编辑器中测试，打开 `Scene2_city_v2` 场景后点击 Play，战斗将在 3 秒后自动开始。

### 3.3 核心玩法系统实现

#### 3.3.1 多路步兵方阵与阵型维护（SquadCommander）

**文件**：`Assets/Script/armymanage/SquadCommander.cs`

`SquadCommander` 负责指挥一个小队的行进、阵型维护和到达城墙后触发架梯。其核心算法 `MaintainFormation()` 每帧执行三个关键策略：

**① 绝对基准轴（防止转弯甩尾）**

```csharp
// 不直接跟随队长的 transform.forward
// 而是计算队长到目标城墙的方向向量作为"绝对轴向"
Vector3 baseForward = targetWall.position - transform.position;
baseForward.y = 0;
Quaternion stableRotation = Quaternion.LookRotation(baseForward);

// 每个小兵的目标位置 = 队长位置 + 绝对基准轴 × 初始相对偏移
Vector3 targetFormationPos = transform.position + stableRotation * localOffsets[member];
```

这样队长在避障转弯时，方阵整体不会甩成弧线，始终保持朝向城墙的整齐队列。

**② 防抖机制（避免高频寻路）**

```csharp
// 只有当新位置小兵当前目的地偏离超过 1 米才重新 SetDestination
if (Vector3.Distance(member.destination, targetFormationPos) > 1.0f)
    member.SetDestination(targetFormationPos);
```

**③ 追赶机制**

```csharp
// 掉队超过 3 米的小兵临时加速到 1.3 倍个人速度
if (distToTarget > 3f) member.speed = personalSpeeds[member];
else member.speed = baseSpeed;
```

#### 3.3.2 精英死士有限状态机（EnemyAssault）

**文件**：`Assets/Script/armymanage/EnemyAssault.cs`

死士是一个 7 状态的有限状态机（FSM），完整状态流转如下：

```
Idle（初始隐藏）
  │  ActivateAndFollow(commander) — 跟随指挥官行军
  ▼
MarchingToWall（跟随队列行军，agent.speed 同步指挥官速度）
  │  StartAssaultWithTarget(bottomAnchor) — 收到导演指令转向梯子
  ▼
RunningToLadder（冲向梯子底部，stoppingDistance=0.1）
  │  OnTriggerEnter(BottomTrigger) — 踩到梯底碰撞体
  ▼
Climbing（transform.Translate 向上攀爬，agent+rb 禁用）
  │  OnTriggerEnter(TopTrigger) — 到达梯顶
  ▼
ClimbingEnd（翻墙动画 1.5s → 对齐 topAnchor）
  │  agent.enabled = true
  ▼
ChasingPlayer（NavMesh 追逐，忽略 Y 轴判定）
  │  水平距离 ≤ 1.5m
  ▼
Attacking（agent.isStopped，Animation Event 触发生效判定）
  │  HitPlayer() → 掉落武器 → 跳转结算
  ▼
（游戏结束）
```

关键设计点：
- **水平距离判定**：所有判定均忽略 Y 轴，防止 VR 玩家身高差异导致判定失效
- **武器拖尾**：Animation Event 精准同步 `EnableWeaponTrail()` / `DisableWeaponTrail()`
- **保底触发**：距离梯子足够近但 OnTriggerEnter 未触发时，直接强行进入攀爬状态

#### 3.3.3 全局阻尼系统（SiegeManager）

**文件**：`Assets/Script/managers/SiegeManager.cs`

阻尼系统是连接玩家命中率与生存压力的核心纽带：

```csharp
// 玩家击杀敌人 → TriggerSuppressingFire()
public void TriggerSuppressingFire()
{
    globalDamping = 0.1f; // 瞬间压制到 10%
}

void Update()
{
    // 推进进度受阻尼影响
    float currentSpeed = baseAdvanceSpeed * globalDamping;
    siegeProgress += currentSpeed * Time.deltaTime;

    // 阻尼缓慢恢复（0.5/s）
    if (globalDamping < 1f)
        globalDamping += dampingRecoveryRate * Time.deltaTime;
}
```

```
玩家击杀敌军 → globalDamping = 0.1 → 全军速度 × 0.1
                                        ↓ 阻尼以 0.5/s 恢复
                                  压力回升 ← 玩家需要再次击杀
```

受阻尼影响的所有消费方：
- `SquadCommander` 每帧：`agent.speed = baseSpeed × siegeManager.globalDamping`
- `EnemyHorse` 每帧：`agent.speed = chargeSpeed × siegeManager.globalDamping`

#### 3.3.4 箭雨物理系统（ArrowRainVolley）

**文件**：`Assets/Script/armymanage/ArrowRainVolley.cs`

每波 200 支箭，12~18 秒触发一波，完整流程：

1. **确定落点**：在 `skyCloudBox` 范围内随机采样 XY 坐标 → 射线检测命中 `wallLayer`
2. **确定起点**：在 `sourceFiringArea` BoxCollider 范围随机采点
3. **计算弹道**：已知起点、终点、飞行时间 1.8 秒，反推初速度

```csharp
Vector3 CalculateLaunchVelocity(Vector3 start, Vector3 target, float time)
{
    Vector3 distance = target - start;
    Vector3 distanceXZ = distance; distanceXZ.y = 0;
    float velocityXZ = distanceXZ.magnitude / time;
    float velocityY = (distance.y - 0.5f * Physics.gravity.y * time * time) / time;
    Vector3 result = distanceXZ.normalized * velocityXZ;
    result.y = velocityY;
    return result;
}
```

4. **对象池管理**：Awake 时预生成 250 支箭 → 运行时从 Queue 取用/归还

### 3.5 渲染管线与 Shader 开发

#### 3.5.1 总体渲染策略

使用 **URP** 确保 PICO 3 移动端性能。采用 **NPR 非真实感渲染** 实现"黑、白、红"水墨风格。不使用复杂后处理栈，通过 Shader 本身完成视觉效果，降低 GPU fillrate。

#### 3.5.2 水墨透明效果（InkCodeShader.shader）

**文件**：`Assets/shader/InkCodeShader.shader`

```hlsl
// 核心：红色通道反相作为透明度
// 白色背景 → alpha = 0（完全透明）
// 黑色墨迹 → alpha = 1（不透明）
half4 frag(Varyings IN) : SV_Target
{
    half4 texColor = SAMPLE_TEXTURE2D(_MainTex, sampler_MainTex, IN.uv);
    half alpha = 1.0 - texColor.r;
    return half4(_InkColor.rgb, alpha * _InkColor.a);
}
```

使用 `Blend SrcAlpha OneMinusSrcAlpha` 标准透明度混合，关闭深度写入（`ZWrite Off`）。

**厚涂水墨变体（InkCodeShaderthick.shader）**：增加 `_Cutoff` 边缘锐度和 `_Opacity` 整体不透明度，使用 `smoothstep` 做硬边裁切。

#### 3.5.3 旗帜飘动（FlagWind / InkFlagWind）

顶点动画 Shader，核心公式：

```hlsl
// 基础版（CG，兼容 Built-in）
float wave = sin(_Time.y * _WindSpeed + (v.vertex.x + v.vertex.y) * _Frequency) * _Amplitude;
v.vertex.xyz += v.normal * wave * v.uv.x;

// InkFlagWind（HLSL + URP）增加世界坐标错帧
float3 worldPos = TransformObjectToWorld(IN.positionOS.xyz);
float phase = worldPos.x * _Frequency + worldPos.y * _Frequency * _Stagger;
float wave = sin(_Time.y * _WindSpeed + phase) * _Amplitude;
```

世界坐标错帧机制使不同位置的旗帜飘动相位不同，避免视觉同步。

### 3.9 性能优化

针对 PICO 3 移动端：

1. **对象池**：箭雨 250 支预生成，Queue 取用/归还，避免 GC 抖动
2. **按需禁用 NavMesh**：SquadCommander 到达城墙后 `agent.enabled = false`；骑兵抵达城门直接 Destroy；工具人 3 秒销毁；箭矢 4~7 秒回收
3. **轻量手写 Shader**：不涉及法线贴图、阴影、反射，仅贴图采样 + 简单数学运算
4. **FFR（Fixed Foveated Rendering）**：PICO 功能，降低边缘像素着色密度
