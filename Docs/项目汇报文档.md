# 《孤城》(The Isolated Fortress) — 项目汇报文档

> **基于具身交互的 VR 历史仿真体验**
> 开发平台：Unity 2022 LTS + PICO 3 (移动 VR) | 核心开发团队：2 人
> 历史背景：东汉永平十八年（公元 75 年），耿恭疏勒城保卫战 — "十三人归玉门"

---

## 一、项目概述与历史背景

### 1.1 历史背景

《孤城》取材自东汉永平十八年（公元 75 年）的真实历史事件——**耿恭疏勒城保卫战**，记载于《后汉书·耿恭传》。

- **东汉戊己校尉耿恭** 率数百汉军在 **疏勒城**（今新疆昌吉回族自治州奇台县附近，天山北麓）抵御 **北匈奴数万大军**
- 疏勒城地处天山北麓、车师前王国境内，是东汉经略西域的咽喉要冲，城旁有涧水可饮
- 匈奴围城，截断涧水，耿恭城中掘井 **十五丈不得水**，吏士渴极，**榨马粪汁饮之**
- 耿恭整衣拜井，泉涌出，匈奴以为有神助，退兵——此后成为著名典故 **"耿恭拜井"**
- 匈奴围城经年，城中 **粮尽，煮铠甲弓弩，食其筋革**，绝不投降
- 匈奴单于以 **"降则封白屋王、妻以女子"** 诱降，耿恭诱杀匈奴使者， **燃其尸于城上**——绝其降路
- 汉章帝建初元年（公元 76 年），司徒鲍昱力排众议，言"今使人于危难之地，急而弃之，外则纵蛮夷之暴，内则伤死难之臣"，章帝遂发张掖、酒泉、敦煌三郡及鄯善兵 **七千余人** 出玉门关救援
- 援军至疏勒，城中仅余 **二十六人**，随援军且战且退，至玉门关时 **仅剩十三人**——"吏士素饥困，发疏勒时尚有二十六人，随路死没，**三月至玉门，唯余十三人**"
- 中郎将 **郑众** 亲为十三人沐浴更衣，上疏盛赞："恭之节义，古今未有"
- 时人称其 **"节过苏武"**——苏武持节十九年，而耿恭在绝境中坚守不屈，以攻为守，气节之壮烈不在苏武之下

> 耿恭以 **不满百人之孤军，抗数万之强敌，守城经年，杀敌数千**，最终以十三人归玉门——这段历史堪称中国军事史上最悲壮的一页。清人赞曰："**疏勒孤城，玉门残卒，凛凛生气，千载如生**"。

这段历史是东汉边疆防御体系中极具代表性的英雄史诗，承载着 **忠勇报国、不屈不挠** 的精神内核。疏勒城遗址（今奇台县石城子遗址）已被列为 **全国重点文物保护单位**，考古发掘证实了汉代城址的规模与布局。

### 1.2 选取意义

| 维度 | 说明 |
|------|------|
| **历史教育** | 将"十三人归玉门"这一边疆军事史实转化为可感知的沉浸体验，弥补传统历史教学的直观性不足 |
| **情感共鸣** | 通过 VR 的具身交互让体验者在体能消耗与紧张感中体会守军的绝望与坚守 |
| **文化传承** | 以交互技术激活东汉边疆历史记忆，不同于单纯的说教式历史传播 |

### 1.3 核心愿景

> **"抛弃摇杆，重回公元 75 年的疏勒城"**

通过 **零遥杆位移、真实物理运动** 与极简的 **黑白红水墨视觉风格**，构建高压的攻城防御体验。

## 二、游玩体验

### 2.1 场景流程

整个体验由 4 个场景串联：启动加载 → 主菜单 → 唯一战斗场景 → 结算。

游戏启动后 BootLoader 自动从 BootScene 跳转 GameStartMenu。玩家点击主菜单的"开始游戏"后进入 **Scene2_city_v2**——所有核心玩法都集中在这一场景。战斗结算后跳转 GameEnd，页面自动从上向下滚动展示结局文字，之后可返回主菜单。

### 2.2 战斗流程

BattleDirector 在进入战斗场景 3 秒后启动战斗，同步执行四条线：

```
战斗启动
  ├─ 波次生成：第一波步兵方阵从多个出兵点同时向城墙推进
  ├─ 箭雨循环：每 15±3 秒 200 支箭从天空射出
  ├─ 骑兵冲锋：开局 10 匹从侧翼冲城门，之后每 8 秒补充
  └─ 精英死士：隐藏跟随在其中一路指挥官身边
```

玩家在城墙上可以做三件事：

**射击压制**——用大黄弩射击城下的敌军。VR 手柄按键触发后，箭矢从枪口沿前方飞出（AddForce 2000），命中敌人后触发 SiegeManager 的全局阻尼机制：阻尼降至 0.1，所有敌军行军速度 ×0.1，几乎停滞约 2 秒后逐渐恢复。

**躲避箭雨**——敌箭从天空沿抛物线落入城内。若玩家被击中则屏幕闪红，箭插在身上，5 秒后自行消失。

**面对精英死士**——某一波次的指挥官到达城墙后，其携带的云梯在 2 秒内竖起（Lerp 旋转），工具人随机逃散 3 秒后销毁。若这把梯子是导演预设的"精英路线"，隐藏跟随的死士立即激活，沿 FSM 执行：跑向梯底 → 攀爬翻墙 → 跳入城内 → 追逐玩家 → 攻击。

死士攻击命中时，玩家闪红并强制丢掉武器，1.5 秒后跳转结算。

### 2.3 全局阻尼——核心战斗循环

```
玩家击杀敌军 → globalDamping = 0.1 → 全军速度 × 0.1
                                        ↓ 阻尼以 0.5/s 恢复
                                  压力回升 ← 玩家需要再次击杀
```

这套机制的战术意义在于：它把"命中率"和"生存压力"直接挂钩。射得准就能获得喘息窗口，射不准敌军就压到城下。阻尼同时影响 SiegeManager 的推进进度条（siegeProgress += baseSpeed × damping × Δt），进度慢则敌军抵达城墙的速度慢，为玩家争取时间。

阻尼影响的不只是步兵。骑兵每帧也读取 damping 值调整 `agent.speed = chargeSpeed × globalDamping`，所以击杀一个敌人能同时拖慢所有兵种。

## 三、技术架构

### 3.1 整体架构

项目以 **BattleDirector** 为核心的战斗指挥架构，各模块间通过 UnityEvent 和直接方法调用通信：

```
                BattleDirector（战斗总控）
                    │
      ┌─────────────┼─────────────┐
      │             │             │
 SiegeManager   ArrowRain    SquadCommander[]
 (全局阻尼)     (箭雨生成)    (多路方阵)
      │                         │
      └───── 影响行军速度 ──────┘
                                │
                         ┌──────┴──────┐
                      SiegeLadder   EnemyAssault
                      (架梯动画)     (死士 FSM)

    AudioManager（音效池，全局单例）
    PlayerHealth（玩家受伤 → 结算）
    SceneChanger（场景切换）
    XRUIManager（暂停菜单）
```

### 3.2 核心系统职责

| 模块 | 文件 | 职责 |
|------|------|------|
| **战斗导演** | `BattleDirector` | 管理战斗生命周期：波次生成、箭雨定时、骑兵调度、精英死士释放判定 |
| **攻城管理** | `SiegeManager` | 维护推进进度和阻尼值，接收压制信号后阻尼降 0.1，每帧恢复 |
| **小队指挥** | `SquadCommander` | 方阵 NavMesh 寻路、阵型维护、到达城墙触发架梯 |
| **精英死士** | `EnemyAssault` | 7 状态有限状态机，跟随 → 跑向梯子 → 攀爬 → 翻越 → 追逐 → 攻击 |
| **骑兵** | `EnemyHorse` | 从侧翼冲锋到城门消失，每帧读取阻尼调速度，死亡触发压制 |
| **箭雨** | `ArrowRainVolley` | 抛物线弹道 + 射线落点 + 250 对象池循环复用 |
| **大黄弩** | `CrossbowShoot` | 手柄触发，AddForce 物理弹道，5 秒销毁 |
| **音频管理** | `AudioManager` | 8 通道池，PlayAttached 挂载到目标播放，播完自动归还 |
| **玩家受伤** | `PlayerHealth` | 中箭 Mesh 球体闪红，中死士强制丢武器 + 跳转结算 |
| **暂停** | `XRUIManager` | 手柄菜单键暂停，启停 Ray Interactor 操作 UI |
| **架梯** | `SiegeLadder` | 2 秒 Lerp 旋转竖起云梯，完成后通知导演 |

### 3.4 精英死士有限状态机

死士的状态切换由位置触发器和导演指令驱动：

```
                    Idle
                     │  ActivateAndFollow(commander)
                     ▼
             MarchingToWall（跟随指挥官行军，agent.speed 同步指挥官速度）
                     │  StartAssaultWithTarget(bottomAnchor)
                     ▼
              RunningToLadder（跑向梯子底部，stoppingDistance=0.1）
                     │  OnTriggerEnter(BottomTrigger)
                     ▼
                Climbing（transform.Translate 向上攀爬，agent+rb 禁用）
                     │  OnTriggerEnter(TopTrigger)
                     ▼
               ClimbingEnd（翻越垛口动画 1.5s → 位置对齐 topAnchor）
                     │  agent.enabled = true
                     ▼
              ChasingPlayer（NavMesh 追逐，忽略 Y 轴，水平距离判定）
                     │  Vector3(忽略Y).magnitude ≤ attackDistance(1.5m)
                     ▼
                Attacking（agent.isStopped，动画事件触发生效判定）
                     │  HitPlayer() → 水平距离验证（含 0.8m 缓冲）→ onPlayerHit
                     ▼
                  （游戏结束，跳转结算）
```

关键设计：攻击判定采用**水平距离**（忽略 Y 轴），防止因 VR 玩家头部高度差异导致判定失效。HitPlayer 由 Animation Event 在刀刃挥出帧调用，配备 0.8 米的武器前摇缓冲距离。

## 四、渲染管线与 Shader

### 4.1 渲染策略

使用 **URP（Universal Render Pipeline）** 确保 PICO 3 移动端性能，采用 **NPR 非真实感渲染** 实现极简的"黑、白、红"水墨视觉风格。不依赖复杂后处理栈，通过 Shader 本身完成视觉效果，降低 GPU fillrate。

### 4.2 Shader Graph 可视化着色器

**地形着色（SG_Terrain）**——通过 Simple Noise 噪波节点混合深浅两套草地色，UV 输入经 Smoothstep 后驱动 Lerp 系数，实现地形的自然斑驳。积雪效果通过 Snow Thickness 参数控制淡入量，View Direction 与法线点积检测视线边缘区域用于特殊处理。节点流程：

```
UV → Simple Noise → Smoothstep → Multiply → Lerp → BaseColor
                                   ↑            ↑
                            "grass dark"   "grass light"
                            Snow Thickness
```

**道具描边（SG_PropOutline）**——通过法线向量检测模型边缘，Step 节点硬边裁切生成纯色描边（而非平滑渐变），最后用 Lerp 与原色混合。Edge thickness 参数控制描边宽度，Unlit 子目标（不参与光照计算）保证性能。

**其他着色器**包括：戈壁地形专属的 gebishader、淡墨城墙 inkwall、远景山体 mount_far（色彩 + 深度渐变）、基础纯色输出的 basecolor。

### 4.3 手写 HLSL Shader

**水墨透明效果（InkCodeShader）**——这是整个视觉风格的核心。贴图采用白底黑墨设计，着色器将红色通道反相作为透明度：白色区域（背景）alpha=0 完全透明，露出背后场景；墨迹部分 alpha>0 半透明叠加 InkColor。`Blend SrcAlpha OneMinusSrcAlpha` 标准透明度混合：

```hlsl
half alpha = 1.0 - texColor.r;
return half4(_InkColor.rgb, alpha * _InkColor.a);
```

**厚涂水墨变体（InkCodeShaderthick）**——在基础版本上增加 `_Cutoff` 边缘锐度和 `_Opacity` 整体不透明度，使用 smoothstep 做硬边裁切，让墨迹边缘更利落而非模糊过渡：

```hlsl
half alpha = smoothstep(_Cutoff - 0.05, _Cutoff + 0.05, texColor.a);
alpha *= _Opacity;
```

**旗帜飘动（FlagWind / InkFlagWind）**——顶点动画 Shader，让旗帜产生实时鼓动效果。核心公式为 `sin(时间 × 风速 + 顶点位置 × 频率) × 振幅`，沿法线方向偏移顶点，UV.x 控制从旗杆根部（静止）到末梢（最大幅度）的渐变：

```hlsl
// CG 版本（FlagWind），兼容 Built-in
float wave = sin(_Time.y * _WindSpeed + (v.vertex.x + v.vertex.y) * _Frequency) * _Amplitude;
v.vertex.xyz += v.normal * wave * v.uv.x;
```

InkFlagWind 在 HLSL + URP 版本中增加了世界坐标错帧机制——将物体的世界坐标 X 和 Y 以不同权重计入相位计算，即使使用同一材质、同一参数，不同位置的旗帜飘动相位也不同，避免视觉同步：

```hlsl
float3 worldPos = TransformObjectToWorld(IN.positionOS.xyz);
float phase = worldPos.x * _Frequency + worldPos.y * _Frequency * _Stagger;
float wave = sin(_Time.y * _WindSpeed + phase) * _Amplitude;
```

### 4.4 材质映射

场景中所有物体按视觉风格分为三类：

- **水墨透明类**（InkCodeShader 系列）——城墙、房屋、部分道具使用白底黑墨贴图，呈现淡墨半透明效果。城墙用 mat_inkwall 表现久经风霜的墨色，房屋用 mat_house 呈现水墨村落感
- **描边类**（SG_PropOutline 系列）——角色单位（outline_enemy、outline_horse、outline_shootman、outline_toolman）和可交互物体（outline_arrow、outline_crossbow、outline_house）使用法线检测描边，赋予物体"手绘感"轮廓
- **旗帜类**（FlagWind / InkFlagWind）——水墨风格旗帜（InkFlag）和普通布质旗帜（Flag_face）在场景中同时飘动，配合世界坐标错帧实现自然异步

### 4.5 环境特效

- **雪花**（VFX Graph）持续从天空飘落，覆盖整个战场

## 五、交互与战斗系统详解

### 5.1 VR 交互框架

项目基于 Unity XR Interaction Toolkit + PICO Unity Integration SDK v3.1.0，核心交互点有三个：

**大黄弩射击**——玩家扣下手柄扳机，CrossbowShoot.Fire() 在 firePoint 位置生成箭矢预制体，对其 Rigidbody 施加 shootForce=2000 的瞬时力。箭矢独立携带一个 AudioSource 播放射击音效，5 秒后自动销毁。

**粘性抓取**——场景中所有武器均使用 StickyGrabInteractable（继承 XRGrabInteractable）。它重写了松手逻辑：玩家松开 grip 键后不会触发掉落，武器依然保持在手中。同时自动设为手柄的子级，消除 VR 中常见的抓取物抖动问题。当精英死士命中玩家时，PlayerHealth 调用 ForceDrop() 强制释放武器，模拟"被攻击震飞"的叙事效果。

**暂停菜单**——PICO 手柄的菜单键触发 XRUIManager 切换暂停状态。暂停时 Time.timeScale=0，启用双手的 Ray Interactor 让玩家通过射线点击 UI 面板，面板出现在视野前方 1 米处。恢复时关闭 Ray Interactor 回到正常 gameplay。

### 5.2 阵型维护算法

多路步兵方阵在向城墙推进时，SquadCommander.MaintainFormation() 每帧执行三个关键策略：

**绝对基准轴**——不直接跟随队长的 transform.forward，而是计算队长到目标城墙的方向向量作为"绝对轴向"。每个小兵的目标位置 = 队长位置 + 该绝对轴向 × 初始相对偏移。这样队长在避障转弯时方阵不会被甩成弧线，始终保持朝向城墙的整齐队列。

**防抖机制**——如果小兵当前的 NavMesh 目的地与新计算的目标位置偏离在 1 米以内，不重新 SetDestination，避免每帧高频寻路。

**追赶机制**——掉队超过 3 米的小兵临时加速到个人速度的 1.3 倍，快速归位。

### 5.3 箭雨物理系统

箭雨每 12~18 秒触发一波，每波 200 支箭。整个流程如下：

1. **确定落点**——在 skyCloudBox 范围内随机采样 XY 坐标，从该点向下发射射线，命中 wallLayer 的 hit.point 即为落点，最多尝试 10 次
2. **确定起点**——在 sourceFiringArea BoxCollider 范围随机采点
3. **计算弹道**——已知起点、终点、飞行时间 1.8 秒，`CalculateLaunchVelocity()` 反推初速度：

   ```
   velocityXZ = horizontalDistance / time
   velocityY = (verticalDistance - 0.5 × gravity × time²) / time
   ```

4. **对象池管理**——Awake 时预生成 250 支箭存入 Queue，运行时从池取出、设置速度、激活、射出。每帧箭矢做 `transform.rotation = LookRotation(rb.velocity)` 保持箭头指向飞行方向。碰撞时 isKinematic=true 插在墙面上，4~7 秒后回池

每支箭射出时通过 AudioManager.PlayAttached() 从 8 通道池借一个 AudioSource 挂到箭上，spatialBlend=0.6 让破空声有空间定位随箭飞行，播完后自动解绑归还池。

### 5.4 全局阻尼系统的技术实现

阻尼系统的核心在 SiegeManager.Update() 中：

```csharp
// 推进进度受阻尼影响
float currentSpeed = baseAdvanceSpeed * globalDamping;
siegeProgress += currentSpeed * Time.deltaTime;

// 阻尼缓慢恢复
if (globalDamping < 1f)
    globalDamping += dampingRecoveryRate * Time.deltaTime;
```

当 EnemyHealth.Die() 或 EnemyHorse.Die() 调用 `TriggerSuppressingFire()` 时，damping 瞬间置为 0.1。受影响的消费方包括：

- SquadCommander.Update() 每帧：`agent.speed = baseSpeed × siegeManager.globalDamping`——队长速度被压制，整个方阵停在原地
- EnemyHorse.Update() 每帧：`agent.speed = chargeSpeed × siegeManager.globalDamping`——骑兵也被压制

### 5.5 玩家受伤系统

中箭（OnTriggerEnter + EnemyArrow Tag）：
- EnsureDamageFlashMesh() 创建一个面向内的球体 Mesh 包围摄像机，使用 PXR_Fade Shader（ZTest Always + Alpha Blend），0.25 秒内从 60% 透明度渐减到 0
- 箭矢 SetParent 挂在玩家 Transform 下、Rigidbody isKinematic=true，TrailRenderer 关闭
- 5 秒后 Destroy

被精英死士攻击（HitPlayer → onPlayerHit UnityEvent）：
- 同样闪红 + ForceDrop 丢武器
- 延迟 1.5 秒后 SceneChanger.LoadGameEndScene()

## 六、动画系统

精英死士的动画通过 Animation Event 驱动逻辑，实现攻击判定与武器特效的精确同步：

**攻击动画**（死士/Attack.anim）包含三个关键帧事件：
- `EnableWeaponTrail()`——武器挥出时开启 TrailRenderer，产生墨迹拖尾效果
- `HitPlayer()`——刀刃到达命中位置时调用，此时做水平距离验证，通过则触发 onPlayerHit
- `DisableWeaponTrail()`——收刀时关闭拖尾

**翻墙动画**（Climbing Up Wall FBX）配合 ClimbingEnd 状态使用，1.5 秒播放完成后将死士位置对齐到城墙顶部的 topAnchor，同时恢复 agent.enabled 开始 NavMesh 追逐。

**骑兵与士兵动画**受 Animator Controller 驱动，包含行军（Walk/Run）、待机（Idle）和死亡（Dead）状态切换。Horse_move.controller 负责马匹四足运动，AC_move.controller 和 enemr_AC.controller 负责步兵的移动与战斗。

## 七、性能优化

针对 PICO 3 移动端的性能约束，优化集中在"减少 CPU 寻路压力和 GPU fillrate"两个方向：

**对象池降低实例化开销**——箭雨系统是最高频的创建/销毁场景。Awake 时预生成 250 支箭，运行时从 Queue 取用和归还，避免 Instantiate/Destroy 的 GC 抖动和 CPU 峰值。

**按需禁用 NavMesh**——每个 SquadCommander 到达城墙后立即 `agent.enabled = false`，不再参与寻路计算。骑兵抵达城门直接 Destroy。工具人 3 秒销毁。死亡小兵 2 秒销毁。箭矢 4~7 秒回收。这些策略确保场上活跃的 NavMeshAgent 数量被严格约束在兵力上限以内（步兵 30 + 骑兵 15）。

**轻量手写 Shader 降低 fillrate**——不使用 URP 的标准光照计算，所有自定 Shader 仅通过贴图采样和简单数学运算实现视觉效果。水墨透明 Shader 不涉及法线贴图、阴影、反射等开销。

**URP 管线 + FFR**——URP 本身为移动端进行了批量渲染优化，配合 PICO 的 Fixed Foveated Rendering 降低边缘像素着色密度。

## 八、价值延伸

项目在核心体验之外，具备以下应用场景：

**数字文旅**——为博物馆、游客中心提供轻量化 VR 历史体验方案。疏勒城遗址（石城子遗址）作为全国重点文物保护单位，结合 VR 体验可让静态遗址"活"起来。

**体感教育**——将"十三人归玉门"的历史叙事融入高沉浸感的身体交互中，不同于传统的被动听讲式历史教学，通过 VR 的具身认知加深记忆。

**全民健身**——核心交互要求动用上肢力量（举弩射击、举盾格挡），配合真实下蹲躲避箭雨，3-5 分钟流程可达中高强度间歇运动效果。

## 九、遇到的问题

### 8.1 PICO SDK 版本兼容性问题

项目打包后在 PICO 设备上无法打开 APK，点击图标后闪退或直接无响应。排查过程如下：

1. **尝试修改 PICO 项目配置**——调整 PXR_ProjectSetting 中的各项开关、API 等级等参数，无效
2. **用简单空场景打包测试**——排除项目自身代码冲突的可能，空场景同样无法打开，说明问题不在业务逻辑
3. **更换 SDK 其他版本号**——尝试其他中间版本，仍无法打开
4. **查阅教程**——看到Bilibili教程使用 **3.1.0 版本**在 Unity 2022 下可正常导出并运行

最终将项目中的 PICO Unity Integration SDK 从 **3.4.0 降级为 3.1.0**，重新打包后 APK 在 PICO 3 上正常打开运行。至此确定原因为 **SDK 3.4.0 与当前 Unity 版本或项目配置存在兼容性问题**
